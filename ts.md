# Техническое задание: универсальный установщик и откатчик `skills`

## 1. Роль
Ты - senior Windows automation engineer / DevOps-архитектор, который глубоко понимает PowerShell, NTFS-ссылки, иерархию пользовательских профилей Windows, идемпотентные установщики, безопасный rollback и автоматизацию для CLI-инструментов.

## 2. Суть
Нужно создать набор скриптов, который централизует папку `skills` для нескольких AI-agent инструментов. Скрипты должны:
- сохранить существующие локальные папки `skills` в резервную копию `skills-backup`;
- заменить их ссылкой на единый источник навыков;
- работать только по заранее известным местам установки;
- запускаться повторно без поломки уже обработанных путей;
- иметь обратимый откат, который возвращает всё в исходное состояние.

Цель - получить безопасное, повторяемое и понятное решение для управления skills во всех поддерживаемых инструментах без ручного переноса файлов.

## 3. Задача
- [ ] Проанализировать и зафиксировать список известных мест, где у AI-agent инструментов может лежать папка `skills`.
- [ ] Спроектировать универсальный установочный скрипт, который принимает путь к исходной папке skills как параметр.
- [ ] Для каждого найденного известного места:
  - [ ] проверить, существует ли целевая папка `skills`;
  - [ ] если папка уже является корректной ссылкой на нужный источник, ничего не менять;
  - [ ] если папка является обычной директорией, перенести её содержимое в `skills-backup`;
  - [ ] заменить `skills` на ссылку на переданный источник;
  - [ ] записать понятный лог выполненных действий;
  - [ ] не трогать пути, которые не входят в список известных и разрешённых мест.
- [ ] Обеспечить повторный запуск скрипта:
  - [ ] новые обнаруженные места должны обрабатываться;
  - [ ] уже обработанные места должны пропускаться без ошибок;
  - [ ] смена исходной папки должна корректно обновлять ссылки.
- [ ] Сделать отдельный скрипт-обёртку для конкретного пути `c:\Users\ivanm\Documents\MD\Technology\iam-skills\skills`, который просто вызывает универсальный установочный скрипт с этим параметром.
- [ ] Сделать отдельный скрипт отката:
  - [ ] удалить созданные ссылки;
  - [ ] восстановить исходные папки из `skills-backup`;
  - [ ] удалить или оставить резервные копии по явно заданному правилу;
  - [ ] быть безопасным при повторном запуске.
- [ ] Подготовить проверку на безопасных тестовых папках перед финальным применением.
- [ ] Подготовить итоговую документацию по использованию, ограничениями и порядку запуска.

## 4. Контекст
- Решение предназначено для Windows-окружения.
- Основной источник навыков: `c:\Users\ivanm\Documents\MD\Technology\iam-skills\skills`.
- В списке ожидаемых целевых инструментов есть:
  - Claude Code
  - Windsurf
  - Codex
  - GitHub Copilot
  - Gemini CLI
  - OpenCode
  - Antigravity
- Важен сценарий повторного запуска: скрипт должен уметь находить новые места установки и не ломать уже обработанные.
- Важно учитывать, что в некоторых местах `skills` уже может быть ссылкой, а в некоторых - обычной папкой с данными.
- Нужно ориентироваться на безопасную работу с пользовательскими данными и не затрагивать чужие каталоги.

## 5. Формат
- Результат должен состоять минимум из 3 скриптов:
  - универсальный скрипт установки;
  - тонкий скрипт-обёртка для конкретного исходного пути;
  - скрипт отката.
- Скрипты должны быть пригодны к запуску из PowerShell на Windows.
- Для каждого скрипта нужен понятный вывод:
  - что найдено;
  - что изменено;
  - что пропущено;
  - что восстановлено.
- Дополнительно нужны:
  - краткая инструкция по запуску;
  - описание поддерживаемых мест;
  - описание поведения при повторном запуске;
  - описание rollback-сценария.

## 6. Ограничения
- Нельзя менять пути вне списка известных и явно поддерживаемых локаций.
- Нельзя делать необратимые действия без резервной копии.
- Нельзя хардкодить исходную папку внутри универсального скрипта - она должна передаваться параметром.
- Нельзя ломать уже существующие корректные ссылки.
- Нельзя удалять пользовательские данные без явного и безопасного механизма восстановления.
- Нельзя рассчитывать на одноразовый запуск - решение должно быть идемпотентным.
- Нельзя завершать работу без проверки на тестовых, безопасных папках.
