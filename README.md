# Skills Unifier

Единый способ управлять папкой `skills` для нескольких AI-инструментов без ручного копирования, дублирования и потери актуальности.

## Проблема

У каждого агента или CLI-инструмента свой локальный путь к `skills`. В итоге:

- навыки быстро расходятся по версиям;
- обновления приходится делать вручную в нескольких местах;
- сложно понять, где лежит актуальный источник;
- при переезде или восстановлении окружения легко сломать структуру;
- откат изменений становится рискованным и долгим.

## Решение

Проект предполагает набор PowerShell-скриптов, которые:

- находят только заранее известные и разрешенные места установки;
- сохраняют существующую папку `skills` в `skills-backup`;
- заменяют локальную папку ссылкой на единый источник;
- работают повторно без поломки уже обработанных путей;
- умеют откатывать изменения обратно в исходное состояние.

## Что должно получиться

- универсальный установочный скрипт с параметром пути к исходной папке `skills`;
- тонкая обертка для конкретного пути `c:\Users\ivanm\Documents\MD\Technology\iam-skills\skills`;
- отдельный скрипт отката;
- безопасная проверка на тестовых папках перед финальным применением;
- документация по запуску, ограничениям и rollback-сценарию.

## Принципы

- Идемпотентность: повторный запуск не должен ломать уже настроенные места.
- Безопасность: не затрагивать неизвестные каталоги и всегда иметь резервную копию.
- Прозрачность: выводить понятный лог, что найдено, что изменено и что пропущено.
- Обратимость: каждый шаг должен иметь понятный rollback.
